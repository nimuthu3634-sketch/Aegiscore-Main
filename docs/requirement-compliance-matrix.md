# AegisCore Requirement Compliance Matrix

This matrix maps the **original proposal requirement set** to the **final product** implementation and validation posture. The **final product** is **AegisCore** itself—the **centralized SOC platform MVP** submitted as the end deliverable (see **[final-product.md](final-product.md)** for the full definition).

**Product positioning:** **AegisCore** is the **delivered application**: a **single-tenant** SOC platform MVP with **centralized monitoring**, **alert triage**, **incident handling**, **explainable AI-assisted risk scoring** (deterministic baseline + optional **TensorFlow/Keras**), **reporting**, and **controlled automated response**. The **academic release** is **transparently limited** to **`brute_force`**, **`port_scan`**, **`file_integrity_violation`**, and **`unauthorized_user_creation`**; the matrix does **not** imply broader threat catalogues, full enterprise feature parity, or multi-tenant production completeness.

**Approved scope (current academic release):** monitor system logs (Wazuh) and network traffic (Suricata); detect only **`brute_force`**, **`port_scan`**, **`file_integrity_violation`**, **`unauthorized_user_creation`**; centralized dashboard; ML-capable risk scoring (baseline default, optional trainable path); basic automated response; incident and response recording; validation via fixtures, automated tests, and documented simulated attacks (optional live connectors in evaluation environments).

**Explicitly beyond this release (documented, not implied):** ransomware/phishing/APT/zero-day product lines; claims of large-scale rollout maturity; and parity with full commercial-grade SOC product suites.

Release-candidate note: this matrix reflects that **transparent MVP** posture—**investor- and examiner-defensible** without overstating production completeness.

## Reviewer Notes

- Requirement compliance is strongest for the **bounded MVP SOC workflow** aligned to the academic release: ingestion, normalization, scoring, incident handling, basic response, and operational reporting.
- Remaining limitations are explicit—primarily **connector profile breadth** and **features intentionally deferred** to preserve a defensible, testable submission.
